Hardware-First Integration Plan

This repository does not contain the ROS2 sovereign_mapper package, so ROS2 C++ source changes are not applied here. The plan is captured as an implementation guide for execution in the actual sovereign_mapper workspace.

Execution Order

  1. Integrate px4_offboard_bridge.cpp first (trajectory output to MAVLink/PX4 setpoints).
  2. Add runtime safety gates (arming/mode checks, watchdog timeout, command-rate guarantees).
  3. Validate planner-to-PX4 end-to-end timing under load.
  4. After control-path stability, add UDP dashboard telemetry pipeline.
  5. Tune telemetry schema/rate so visualization never degrades flight-control latency.

px4_offboard_bridge.cpp integration checklist

  • Create bridge node subscribing to planner trajectory output and publishing PX4 offboard setpoints.
  • Add strict timestamp freshness checks and reject stale trajectories.
  • Enforce minimum publish rate for offboard mode continuity.
  • Add mode/arming state validation before sending motion setpoints.
  • Implement watchdog fallback to hover/hold if trajectory updates stop.
  • Log command latency metrics for planner output to PX4 ingestion path.
  • Run staged validation: SITL first, then tethered/guarded hardware tests.
